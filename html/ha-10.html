    pacemaker

    プロセス： heartbeat:master contrall process heartbeat:FIFO reader heartbeat:write bcast ethX heartbeat:read bcast ethX ... ccm cib [cib=XML形式の・・・] lrmd -r attrd crmd stonithd command: crm crm_attribute Manage node's attributes and cluster options. crm_failcount A convenience wrapper for crm_attribute crm_mon Provides a summary of cluster's current state. crm_node Tool for displaying low-level node information crm_resource Perform tasks related to cluster resources. crm_shadow Perform configuration changes in a sandbox before updating the live cluster. crm_standby A convenience wrapper for crm_attribute crm_uuid crm_verify Check a (complete) confiuration for syntax and common conceptual errors. crmadmin Development tool for performing some crmd-specific commands.

    セットアップ

    Linux系ディストリビューションCentOSでPacemakerを使う場合、 Linux-HA-Japanプロジェクト(http://linux-ha.sourceforge.jp/wp/)が提供しているパッケージを インストールするのが最も簡単、且つ確実である。 

    http://sourceforge.jp/projects/linux-ha/releases/?package_id=11413
    pacemaker-1.0.11-1.2.1.el5.x86_64.repo.tar.gz 
    root> mv pacemaker-1.0.11-1.2.1.el5.x86_64.repo.tar.gz /tmp
root> cd /tmp
root> tar zxvf pacemaker-1.0.11-1.2.1.el5.x86_64.repo.tar.gz
root> cd pacemaker-1.0.11-1.2.1.el5.x86_64.repo
root> yum -c pacemaker.repo install corosync.x86_64 heartbeat.x86_64 pacemaker.x86_64

    ソースコードから全てを構築するには、以下のサイトを参照。 

    http://www.linux-ha.org/wiki/Download/ja

        * /etc/ha.d/ha.cf ファイルを以下のように編集します

pacemaker on
logfacility local1

debug 0
udpport 694       ← 複数のクラスタを同じインターコネクトLAN上で使用する場合は
                    ポート番号を変更すること 
keepalive 2
warntime 20
deadtime 24
initdead 48

bcast eth1        ← 1本目のインターコネクトのインタフェース
bcast eth3        ← 2本目のインターコネクトのインタフェース

node pm01         ← Heartbeat3で使用する1台目のサーバの「uname -n」コマンドで表示されるホスト名
node pm02         ← Heartbeat3で使用する2台目のサーバの「uname -n」コマンドで表示されるホスト名
watchdog /dev/watchdog
